Adding new event in orchestrator microservice

Hi everyone, in this blog I'll be sharing how we added new event in our orchestrator microservice.
Context:
Let's assume we have a microservice [Service-1] which interacts with other microservices using events. Now Service-1 sends request to Service-2 to get some data, say record details for given user and when data is retrieved and sent to Service-1 successfully, Service-2 produces an event [Event-step-completed] which is consumed by Service-1. Since response from Service-2 requires a lot of time, we now had to implement a new business use case were we need to produce a new event [Event-step-not-completed] from Service-1 when time-limit exceeds which is then consumed by the Service-1.
Kafka is primarily used to produce and consume these events in these microservices.
We created the new event [Event-step-not-completed] using pre-existing event interface and added logic to generate event from Service-1 after pre-defined time-limit exceeds. Then, we checked for the all the topics consumed by the consumers in Service-1 and assigned event [Event-step-not-completed] to topic [Topic-step-not-completed]. We added logic in the consumer of Service-1 to consume event [Event-step-not-completed].
After creating the event [Event-step-not-completed] for the Service-1, we found that
  • We have improved the response time for Service-1 as we only wait for response from Service-2 for pre-defined time-limit after which we produce Event-not-completed.
  • We can further implement event [Event-step-not-completed] for other steps in our microservice [Service-1] to create a check-list of completed steps,which can be used to identify the step where our orchestrator microservice [Service-1] fails.
  • We can also add a new functionality to resume any process in Service-1 from the last step which is completed in the check-list.
